Laboratory Progression (Labs 1–5)

Development journey tracking from static layouts to persistent local repository integrations.

Lab 1

Declarative UI Layout Architecture

Core Focus: Structural layout orchestration utilizing Jetpack Compose primitives (Row, Column, Box), explicit Modifier chaining, and layout constraints.

Technical Synthesis: Shifted from imperative XML to declarative composition trees. Reverse-engineered UI mockups into dynamic layout structures to form the visual baseline for the OnRoad application.

Lab 2

Reactive State Management

Core Focus: Implementation of runtime tracking via remember { mutableStateOf() }, click listeners, and state hoisting patterns.

Technical Synthesis: Created interactive text input fields and buttons. Prevented redundant recompositions by hoisting transient state properties, ensuring efficient UI updates and reliable event handling pipelines.

Lab 3

Material Design System Customization

Core Focus: Defining cohesive Material Theme color tokens, custom typography scales, programmatic shapes, elevations, and dynamic card transitions using animateContentSize().

Technical Synthesis: Customized application themes to enforce uniform branding. Utilized animated Card containers to cleanly structure and toggle dense pieces of transit information.

Lab 4

Navigational Routing Graphs

Core Focus: Decoupled application navigation handled via NavHostController, centralized screen state management within Jetpack ViewModels, and safe type-parameter routing.

Technical Synthesis: Restructured the app into an MVVM architecture. Isolated UI presentation from state tracking to ensure the user's data persists cleanly through device configuration changes.

Technical Synthesis: Built an offline-first database framework. Structured transactional queries to read, write, and safely stream local travel entities without interrupting user interactions.

Project Milestones

Project Phase 1

OnRoad V1: Architectural Core & Navigational Foundation

Targeting SDG Target 11.2: Accessible, safe, and sustainable public transport infrastructures.

Problem Engineering: Urban transit apps often face fragmented information loops and fragile state management, leading to visual data loss or performance degradation during complex screen navigation loops.

Implementation Strategy: Implemented a 5-screen MVVM architectural framework utilizing centralized state handling with Jetpack ViewModels. Established clear navigation pathways while keeping layout layers modular and robust.

Project Phase 2

OnRoad V2: Full Production App (Cloud & Hardware)

Advanced Build: Cloud Database, Device Sensors, and Live Weather API.

The Problem (SDG 11 Extension): Basic transport apps fail without web access or physical context. Modern applications must connect hardware sensors, offline caching, and live cloud syncing to serve users reliably on the road.

Key Features Built

  • 8 Distinct Screens: Created map interfaces, custom profile pages, traffic alert dashboards, and calculation layouts.
  • Room Caching: Added safe, local SQLite storage so the app stays functional without network coverage.
  • Firebase Cloud Database: Connected live cloud synchronization so users can post and read crowdsourced reports instantly.
  • GPS Hardware Integration: Set up native device location tracking using low-level threads to pull real-time coordinates.
  • Fuel & Eco Calculator: Built a tool that calculates fuel use against regional gas prices to encourage sustainable driving habits.

Reflection & Learning Journey

1. Personal Growth & Architectural Development Journey

Advancing from static UI rendering in Lab 1 to building a connected multi-tier architecture in Project 2 redefined my coding workflows. Moving away from localized data layouts toward asynchronous repository pipelines taught me the value of structured data tracking patterns and sound presentation abstractions.

2. Technical Hurdles & Engineering Resolutions

Managing asynchronous database operations alongside remote cloud writes threatened race conditions on the main UI thread. I resolved this by offloading database operations onto background thread contexts via Kotlin Coroutines, exposing state safely through lifecycle-aware StateFlow objects to achieve fluid layout rendering.

3. Sustainable Development Goals (SDG) Societal Impact

The OnRoad application directly aligns with SDG 11. By merging real-time coordinate sensors with an analytical fuel calculator, the app helps road users minimize traffic idling and optimize trip efficiency, addressing city-wide traffic bottlenecks and carbon output concerns.

4. Academic Integrity & AI Transparency Statement

Generative AI utilities were used exclusively as syntax checkers, error logs diagnostics, and design layout inspiration models. All application structures, schemas, database entities, repository architectures, and multi-screen components were designed and managed manually to preserve authentic code ownership and technical agility.
